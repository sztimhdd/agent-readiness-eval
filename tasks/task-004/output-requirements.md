## `answer/task-id.txt`
Exactly: `task-004`

## `answer/final-answer.md`
Must include:
- **Executive Summary**: what the project does, what bugs were found, and that all tests pass after repair.
- **Bug Report**: for each of the 3 bugs, describe the symptom (which test failed), the root cause (what line of code was wrong), and the fix applied.
- **Test Results**: confirm that after repair, `python3 -m unittest discover -s tests -v` shows all tests passing.
- **Reconciliation Summary**: report the number of discrepancies found and their types.

## `answer/decision-log.md`
Contains concise source citations, decisions, verification steps, and declared recovery actions. Must not request or contain hidden chain of thought.

## `answer/artifacts/project/`
A complete copy of the `environment/base-project/` directory with your fixes applied in `src/`. All tests must pass from this copy.

## `answer/artifacts/change-summary.md`
For each of the 3 bugs, describe:
- Which test failed
- The root cause (file, line, why it was wrong)
- The fix applied (what you changed and why)

## `answer/artifacts/reconciliation-report.json`
Copy of `output/reconciliation-report.json` generated by running the repaired `src/reconcile.py`.

## `answer/run-metadata.json`
Copy from `templates/run-metadata.json`. Use literal string `UNAVAILABLE` for any field the harness cannot observe. Do not estimate token counts, timings, or tool calls.
